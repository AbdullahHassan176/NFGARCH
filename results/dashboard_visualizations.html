Research Overview

Research Question: Does replacing fixed innovation assumptions in GARCH models with data-driven residual distributions learned by Normalising Flows (NFs) improve forecasting accuracy, distributional realism, and risk calibration for exchange rate and equity returns?

Methodology

This research employs a two-stage NF-GARCH design:

  1. Stage 1: Fit classical GARCH models (sGARCH, eGARCH, TGARCH, gjrGARCH) to financial returns
  2. Stage 2: Extract standardized residuals, train Normalising Flows (MAF - Masked Autoregressive Flow) on these residuals, and reinject flow samples as innovations into the unchanged volatility recursion
Design Choice: We use a two-stage NF-innovations approach rather than end-to-end "flow-after-scaling" models. This preserves the classic volatility recursion while gaining distributional flexibility in innovations, making estimation modular and robust. Trade-off: the flow's shape is state-invariant (scales with σ_t but doesn't depend on it), whereas end-to-end models can allow conditional shape to vary more richly with volatility.

Dataset

The analysis uses 6 financial assets (optimized from 12 for computational efficiency):

  • Equity Assets (3): NVDA, MSFT, AMZN
  • FX Pairs (3): EURUSD, GBPUSD, USDZAR

Data spans daily returns from August 2005 to August 2024, with a 65:35 chronological train/test split and optimized time-series cross-validation using reduced window sizes.

Evaluation Framework

Models are evaluated using multiple metrics:

  • Forecasting Metrics: MSE, MAE, AIC, BIC, Log-Likelihood
  • Distributional Metrics: KS distance, Wasserstein distance, Tail index, Skewness, Kurtosis
  • Stylized Facts: Volatility clustering, Leverage effects, Autocorrelation decay, Heavy tails
  • Risk Metrics: VaR exceedance rates, Kupiec test, Christoffersen test, Expected Shortfall
  • Stress Testing: Historical crises (2008 GFC, 2020 COVID), hypothetical shocks

1. Model Performance Comparison

Forecasting Accuracy

Model performance is evaluated using Mean Squared Error (MSE) and Akaike Information Criterion (AIC) on out-of-sample test data.

Figure 1: Mean Squared Error by Model (Chronological Split). Lower values indicate better forecasting performance.

Interpretation: Models with lower MSE provide more accurate volatility forecasts. The plot shows relative performance across all GARCH variants, helping identify which models best capture volatility dynamics for different asset classes.

Figure 2: Akaike Information Criterion by Model. Lower values indicate better model fit while penalizing complexity.

AIC Explanation: AIC = 2k - 2ln(L), where k is the number of parameters and L is the likelihood. AIC balances model fit with complexity, favoring models that achieve good fit with fewer parameters. The model with the lowest AIC is preferred (Akaike, 1974).
Reference: Akaike, H. (1974). A new look at the statistical model identification. IEEE Transactions on Automatic Control, 19(6), 716-723.

2. Distributional Metrics

Distributional metrics assess how well synthetic residuals from Normalising Flows match the empirical distribution of actual GARCH residuals.

Kolmogorov-Smirnov Distance

Figure 3: Kolmogorov-Smirnov Distance by Model. Lower values indicate better distributional match.

KS Distance: The KS statistic measures the maximum vertical distance between two cumulative distribution functions. KS distance ranges from 0 to 1, where 0 indicates identical distributions. Lower KS distance means the NF-generated residuals better replicate the empirical residual distribution (Kolmogorov, 1933; Smirnov, 1939).

Wasserstein Distance

Figure 4: Wasserstein Distance by Model (log scale). Lower values indicate better distributional match.

Wasserstein Distance: Also known as Earth Mover's Distance, Wasserstein distance measures the minimum cost to transform one distribution into another. It captures both location and shape differences and is more sensitive to tail behavior than KS distance (Villani, 2009).

Tail Index

Figure 5: Tail Index (Hill Estimator) by Model. Higher values indicate heavier tails.

Tail Index: The Hill estimator quantifies tail heaviness in financial returns. Higher tail index values indicate heavier tails (more extreme events). Financial returns typically exhibit heavy tails (tail index > 2), requiring distributions beyond Gaussian (Hill, 1975).
References:
• Kolmogorov, A. (1933). Sulla determinazione empirica di una legge di distribuzione. Giornale dell'Istituto Italiano degli Attuari, 4, 83-91.
• Smirnov, N. (1939). Estimate of deviation between empirical distribution functions in two independent samples. Bulletin Moscow University, 2(2), 3-16.
• Villani, C. (2009). Optimal Transport: Old and New. Springer.
• Hill, B. M. (1975). A simple general approach to inference about the tail of a distribution. Annals of Statistics, 3(5), 1163-1174.

3. Stylized Facts of Financial Returns

Stylized facts are statistical properties consistently observed in financial time series. Good models should replicate these empirical regularities.

Volatility Clustering

Figure 6: Volatility Clustering Persistence by Asset Class. Higher values indicate stronger clustering.

Volatility Clustering: Large price changes tend to be followed by large changes (of either sign), while small changes tend to be followed by small changes. This is measured via autocorrelation of squared returns. Persistent volatility clustering is a key feature of financial markets that GARCH models were designed to capture (Engle, 1982).

Leverage Effect

Figure 7: Leverage Effect by Asset Class. Positive values indicate negative returns increase future volatility more than positive returns.

Leverage Effect: Negative returns have a larger impact on future volatility than positive returns of the same magnitude. This asymmetry is captured by eGARCH, TGARCH, and gjrGARCH models. The leverage effect is economically motivated: falling asset prices increase leverage ratios, making firms riskier (Black, 1976).
References:
• Engle, R. F. (1982). Autoregressive conditional heteroscedasticity with estimates of the variance of United Kingdom inflation. Econometrica, 50(4), 987-1007.
• Black, F. (1976). Studies of stock price volatility changes. Proceedings of the 1976 Meeting of the Business and Economic Statistics Section, American Statistical Association, 177-181.

4. Value at Risk (VaR) Backtesting

VaR backtesting evaluates whether predicted risk measures match realized losses, ensuring models are properly calibrated for risk management applications.

Exceedance Rates

Figure 8: VaR Exceedance Rates at 95% Confidence Level. Expected rate is 5% (red dashed line).

Exceedance Rate: The proportion of observations where actual losses exceed predicted VaR. For a correctly calibrated 95% VaR model, the exceedance rate should be approximately 5%. Rates significantly above 5% indicate underestimation of risk, while rates below 5% indicate overestimation (Basel Committee, 2011).

Kupiec Test

Figure 9: Kupiec Test P-values. P-values > 0.05 indicate models pass the test (not significantly different from expected rate).

Kupiec Test: A likelihood ratio test for unconditional coverage. H₀: The exceedance rate equals the expected rate. If p-value > 0.05, we fail to reject H₀, indicating proper calibration. This test checks whether the model produces the correct number of exceedances on average (Kupiec, 1995).
References:
• Basel Committee on Banking Supervision. (2011). Basel III: A global regulatory framework for more resilient banks and banking systems. Bank for International Settlements.
• Kupiec, P. H. (1995). Techniques for verifying the accuracy of risk measurement models. Journal of Derivatives, 3(2), 73-84.

6. NF-GARCH vs Standard GARCH Comparison

This section compares the performance of NF-augmented GARCH models against their classical counterparts.

MSE Comparison

Figure 12: Mean Squared Error Comparison: NF-GARCH vs Standard GARCH. Lower is better.

Key Finding: NF-GARCH models typically show lower MSE than standard GARCH models, indicating improved forecasting accuracy. The flexible residual distributions learned by Normalising Flows better capture empirical innovation characteristics, leading to more accurate volatility forecasts.

Asset Class Analysis

Figure 13: Performance Comparison by Asset Class (FX vs Equity).

Asset-Class Findings:
  • Equities: NF-sGARCH typically performs best (parsimonious variance recursion + flexible innovations)
  • FX Pairs: NF-eGARCH typically performs best (log-variance dynamics + flexible innovations)
This suggests that (i) correcting innovation misspecification yields large gains, and (ii) the preferred volatility recursion differs by market microstructure and leverage intensity.
Research Contribution: This analysis demonstrates that distributional flexibility in innovations can significantly improve forecasting performance without altering volatility dynamics. The asset-class dependent preferences suggest market microstructure considerations in model selection.
